# Triage: Issue #3 — lifecycle.ts + hookable integration

## Root cause / context

Issue #3 is a greenfield task (no bug, no regression). The daemon needs a
validated hook-invocation layer that sits between raw `hookable` calls and the
rest of the workflow. Without it, hooks could receive or produce malformed
context objects with no structured error feedback.

### lifecycle.ts design
- `createDaemonHooks()` — thin wrapper around `createHooks<AgenTicaHookMap>()`
  from `hookable`; typed via the `AgenTicaHookMap` interface defined in `types.ts`.
- `hookSchemas` — explicit `Record<keyof AgenTicaHookMap, z.ZodTypeAny>` mapping
  every hook name to its corresponding schema from `schemas.ts`.
- `callStep(hooks, name, ctx)`:
  1. `schema.safeParse(ctx)` — throws `(pre)` error if invalid
  2. Snapshot `ctx` via `serializeCtx` (JSON + `[function]` placeholder)
  3. `await hooks.callHook(name, ctx)` — catches and re-throws with step name
  4. `schema.safeParse(ctx)` — throws `(post)` error if hooks produced invalid ctx
  5. Deep-equal comparison of before/after snapshots → logs `{ step, mutated: true }`

### Test coverage (7 tests, all pass)
