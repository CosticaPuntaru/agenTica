# Triage: Issue #4 — Task: Config loader v3

## Summary

Rewrite `loadConfig()` in `github-daemon.mjs` to:
1. Rename all SCREAMING_SNAKE_CASE config keys to camelCase (clean break, no shim)
2. Support `.agenTica.ts` config files via `tsx/esm` loader (fallback to `@swc-node/register`)
3. Bootstrap the `hookable` hook system via `createDaemonHooks()` from `lifecycle.ts`
4. Call `onStart`/`onStop` lifecycle callbacks

## Root Cause

The daemon was written with SCREAMING_SNAKE_CASE config keys that don't match the v3 TypeScript types
(`AgenTicaConfig` in `types.ts`) or the Zod schemas (`agentConfigSchema` in `schemas.ts`), which both
already use camelCase. The hook system types and lifecycle implementations were added in issues #2 and #3
but `loadConfig()` was never updated to wire them up.

## Fix Plan (TDD)

1. Write tests for the new `loadConfig()` behavior:
   - Returns camelCase keys
   - Loads `.agenTica.ts` via tsx when available
   - Warns and skips `.ts` config when tsx is not available
   - Calls `createDaemonHooks()` and stores result on `CONFIG.hooks`
   - Calls `hooks.addHooks(userConfig.hooks)` when hooks are provided
   - Returns `userConfig` on CONFIG for onStart/onStop access

2. Rewrite `loadConfig()`:
   - Rename all internal defaults to camelCase
   - Add `.agenTica.ts` detection before `.agenTica.js`
   - Integrate `createDaemonHooks()` from `lifecycle.ts`
   - Add `userConfig` to returned CONFIG object

3. Update all CONFIG/agent property accesses in the daemon:
   - `CONFIG.POLL_INTERVAL_MS` → `CONFIG.pollIntervalMs`
   - `CONFIG.LOG_FILE` → `CONFIG.logFile`
   - `CONFIG.LABELS` → `CONFIG.labels`
   - `CONFIG.QUERY(...)` → `CONFIG.buildQuery(...)`
   - `CONFIG.PICKER(...)` → `CONFIG.pickIssue(...)`
   - `CONFIG.IS_BLOCKED(...)` → `CONFIG.isBlocked(...)`
   - `CONFIG.GET_AGENT(...)` → `CONFIG.getAgent(...)`
   - `CONFIG.GET_REVIEW_PROMPT` → `CONFIG.getReviewSkill`
   - `CONFIG.GET_CODE_PROMPT` → `CONFIG.getCodePrompt`
   - `agent.COMMAND` → `agent.command`
   - `agent.LABELS` → `agent.labels`
   - `agent.GET_BASE_BRANCH(...)` → `agent.getBaseBranch(...)`
   - `agent.GET_SKILL(...)` → `agent.getSkill(...)`
   - `agent.GET_MODEL(...)` → `agent.getModel(...)`
   - `agent.ARGS` → `agent.args`

   - `agent.MODEL` → `agent.model`

4. Update `shutdown()` to call `await CONFIG.userConfig.onStop?.({ signal })`

5. Update `main()`:

   - Use new camelCase CONFIG keys in log messages
